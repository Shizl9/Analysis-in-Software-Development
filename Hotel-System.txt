## 1. Analysis Phase

### 1.1 Definition & Importance
The Analysis Phase is the process of gathering, studying, documenting, and validating all required features of the Hotel Management System (HMS).  
It ensures that all stakeholders such as receptionists, guests, managers, and housekeeping staff clearly understand the required system functions.

**Importance:**
- Reduces system errors caused by unclear requirements  
- Helps define clear scope and features  
- Ensures customer satisfaction  
- Forms the base for backlog, design, development, and testing

---

### 1.2 Sub-Steps of the Analysis Phase

#### ‚úî Requirements Elicitation
Collect requirements from:
- Guests  
- Receptionists  
- Hotel managers  
- Housekeeping staff  
- Restaurant staff  
- System administrators

Using:
- Interviews  
- Surveys  
- Observation  
- Document analysis  

#### ‚úî Requirements Analysis
Organize requirements into:
- **Functional Requirements:**  
  Booking rooms, managing guests, payments, housekeeping, restaurant orders  
- **Non-Functional Requirements:**  
  Performance, security, availability, scalability

Identify:
- Conflicts  
- Dependencies  
- Risks  

#### ‚úî Requirements Specification
Document requirements in:
- User Stories  
- Use Case Diagrams  
- Process diagrams (Booking workflow, Check-in/Check-out process)

#### ‚úî Requirements Validation
Review all requirements with stakeholders and get approval.

---
